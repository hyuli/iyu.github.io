Java中的Atomic

2016-05-29
Java Concurrency
Java Concurrency

JDK 5之前是靠Synchronized来保证同步操作的，虽然可以确保正确性，但也带来了问题。

  1. 高并发下吞吐量大量下降。
  2. 上下文切换耗费大量资源。
  3. 可能会造成死锁。

JDK 5利用了现代处理器的CAS(compare and set)原语解决了这个问题。

CAS 操作包含三个操作数 —— 内存位置（V）、预期原值（A）和新值(B)。如果内存位置的值与预期原值相匹配，那么处理器会自动将该位置值更新为新值。否则，处理器不做任何操作。无论哪种情况，它都会在 CAS 指令之前返回该位置的值。（在 CAS 的一些特殊情况下将仅返回 CAS 是否成功，而不提取当前值。）CAS 有效地说明了“我认为位置 V 应该包含值 A；如果包含该值，则将 B 放到这个位置；否则，不要更改该位置，只告诉我这个位置现在的值即可。”
通常将 CAS 用于同步的方式是从地址 V 读取值 A，执行多步计算来获得新值 B，然后使用 CAS 将 V 的值从 A 改为 B。如果 V 处的值尚未同时更改，则 CAS 操作成功。
类似于 CAS 的指令允许算法执行读-修改-写操作，而无需害怕其他线程同时修改变量，因为如果其他线程修改变量，那么 CAS 会检测它（并失败），算法可以对该操作重新计算。清单 3 说明了 CAS 操作的行为（而不是性能特征），但是 CAS 的价值是它可以在硬件中实现，并且是极轻量级的（在大多数处理器中）：

在Mac Mini 2014下安装Ubuntu 16.04

2016-05-25
linux
Install, Ubuntu, linux

家里闲置了一台Mac Mini 2014，自己动手把硬盘换掉成SSD（记得一定要去找那种Mac Mini专用的螺丝刀，普通的不能用，移动主板的铁丝可以自己做，不要去花冤枉钱），就开始装Ubuntu了。Ubuntu支持Mac的UEFI分区，安装十分简单，基本上一次成功。

下载ISO

去Ubuntu官网下载ISO文件，（Ubuntu的镜像速度杠杠的）。http://www.ubuntu.org.cn

Burn镜像到U盘

以下操作是在Mac下操作的，windows请自行搜索。

  1. 用Mac自带工具Disk Utility格式化U盘。

  2. 把ISO转换成img格式,如果生成的文件有dmg后缀，需要去掉。
    $ hdiutil convert -format UDRW -o ubuntu.img ubuntu.iso

  3. 查看当前U盘的盘符，譬如说下面的/dev/disk2
    $ diskutil list

  4. Unmount U盘，记得用上面找到的U盘符
    $ diskutil unmountDisk /dev/disk2

  5. 镜像写入U盘，这个需要花费点时间，这段时间可以去做点其他事情，免得误操作影响了写入，我这里1.4G的镜像写了18分钟多。
    $ sudo dd if=/Downloads/ubuntu.img of=/dev/disk2 bs=1m

执行安装

最后将U盘插入Mac，开机的时候按住Opt键，就进入U盘选择，记得选择UEFI，里面有提示去安装Ubuntu，剩下的过程按照默认来就可以了，比装windows还简单。
